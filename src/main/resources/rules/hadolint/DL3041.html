Rationale :

Version pinning forces the build to retrieve a particular version regardless of whatâ€™s in the cache. This technique can also reduce failures due to unanticipated changes in required packages.

Problematic code :

RUN dnf install -y httpd && dnf clean all

Correct code :

RUN dnf install -y httpd-2.24.2 && dnf clean all

Notes :

If you need downgrade an already installed package to an older version, use dnf downgrade -y <package_name>-<version_info>
