Rationale :

In a Dockerfile, using the useradd command without -l option and a big UID (more than 5 digits) will result in Docker creating a huge log file (several GB). This could break the docker build with a no space left on device error and impact your entire build system.

Problematic code :

RUN useradd -u 99900000 user

Correct code :

RUN useradd -l -u 99900000 user
